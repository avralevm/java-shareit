# java-shareit

## Описание проекта

Shareit - это веб-приложение для обмена вещами. Пользователи могут публиковать объявления о вещах, которые они готовы отдать в аренду, а другие пользователи могут находить эти вещи и запрашивать их использование.

---

## Технологический стек

- **Java 11**: Основной язык программирования.
- **Spring Boot**: Фреймворк для создания веб-приложений.
- **Spring Data JPA**: Для работы с базой данных.
- **PostgreSQL**: Реляционная база данных для хранения данных.
- **Maven**: Система управления зависимостями и сборки проекта.
- **Docker**: Для контейнеризации приложения и базы данных.

---
## Основные возможности:
- Регистрация и аутентификация пользователей.
- Создание, просмотр, редактирование и удаление объявлений о вещах.
- Поиск вещей по названию и описанию.
- Бронирование вещей на определенные даты.
- Просмотр своих бронирований и бронирований своих вещей.

## Как запустить

1.  **Склонируйте репозиторий:**
    ```bash
    git clone https://github.com/username/java-shareit.git
    cd java-shareit
    ```

2.  **Сконфигурируйте приложение:**
    - Основные настройки находятся в файле `application.properties` в директории `server/src/main/resources`.
    - Убедитесь, что у вас установлен и запущен PostgreSQL. Измените `spring.datasource.url`, `spring.datasource.username` и `spring.datasource.password` в соответствии с вашими настройками.

3.  **Соберите и запустите сервер:**
    - Перейдите в директорию `server`:
      ```bash
      cd server
      ```
    - Соберите проект с помощью Maven:
      ```bash
      mvn clean install
      ```
    - Запустите приложение:
      ```bash
      java -jar target/shareit-server-0.0.1-SNAPSHOT.jar
      ```

4.  **Запуск с помощью Docker Compose:**
    - Убедитесь, что у вас установлен Docker и Docker Compose.
    - В корневой директории проекта выполните команду:
      ```bash
      docker-compose up
      ```
    Это запустит и серверную часть, и базу данных PostgreSQL в Docker-контейнерах.

После запуска приложение будет доступно по адресу `http://localhost:9090`.
